# examples/

This directory contains ready-to-use workflow snippets and fixtures for `pr-summary`.

- **`minimal-workflow.yml`** — the smallest possible workflow that runs the action on every PR.
- **`detailed-workflow.yml`** — every input set explicitly, with comments.
- **`multilingual.yml`** — non-English comments (Russian, Japanese).
- **`anthropic-compat.yml`** — using a third-party proxy that speaks the OpenAI protocol in front of Anthropic.
- **`self-hosted-llm.yml`** — pointing the action at a local Ollama or LM Studio server.
- **`sample-diff.patch`** — a realistic 8-file, ~250-line PR diff used to test the action locally and to generate `example-comment.md`.
- **`example-comment.md`** — the actual comment the action would post for `sample-diff.patch`, generated by `npm run examples:comment`.

## How to use the example files

```bash
# Copy any workflow into your repo
cp examples/minimal-workflow.yml .github/workflows/pr-summary.yml

# Regenerate the example comment after editing sample-diff.patch
npm run examples:comment
```

The example workflows assume you have already added `OPENAI_API_KEY` (or another provider's key) as a repository or organisation secret. The action also reads the workflow-provided `GITHUB_TOKEN` automatically — you do not need to mint your own PAT.
